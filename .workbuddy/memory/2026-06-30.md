## 修复 GridViewColumn Width 星号值报错
- **错误**：`无法将"1.5**"字符串转换为 Length` — HistoryView 和 HeaderDatabaseView 两个界面均报错
- **根因**：`GridViewColumn.Width` 是 `double` 类型，不支持 `GridLength` 的星号比例语法（如 `1.5*`、`2*`）
  - `Grid.ColumnDefinition Width="1.5*"` → `GridLength` 类型 ✅ 支持
  - `GridViewColumn Width="1.5*"` → `double` 类型 ❌ 不支持
- **修复**：将所有 GridViewColumn 的星号宽度替换为固定像素值
  - HistoryView：`1.2*`→170, `2*`→283
  - HeaderDatabase Owner：三个 `1.5*`→169, `2.5*`→281
  - HeaderDatabase Customer：`2*`→210, 两个 `1.5*`→158, `2.5*`→263
- **编译**：0 错误

## HistoryView & HeaderDatabaseView 表格重写（ListBox + Grid 方案）
- **动机**：GridView 原生表头无法可靠隐藏（Height=0 Template 残留白色拖动条、ColumnHeaderContainerStyle={x:Null} 无效等）
- **方案**：彻底替换 `ListView`+`GridView` 为 `ListBox`+`VirtualizingStackPanel`+`Grid` ItemTemplate
  - 表头：独立 `Border` + `Grid`（不涉及任何 ListBox 渲染），完全不可拖动
  - 行体：`ListBox` 的 `ItemTemplate` 为 Grid 布局，列宽与表头严格一致
  - `ItemContainerStyle` 覆盖 `ListBoxItem` Template 为纯 `ContentPresenter`，消除选中高亮和焦点虚线
  - 交替行色通过 `AlternationCount="2"` + `DataTrigger` 绑定 `ItemsControl.AlternationIndex` 实现
- **HistoryView.xaml**：5 列（170/283/130/130/175），DataTemplate `QuotationRowTemplate`
- **HeaderDatabaseView.xaml**：保留 Owner/Customer 双表 + 新建弹窗，两个 DataTemplate（`OwnerRowTemplate` 169/169/169/281/100、`CustomerRowTemplate` 210/158/158/263/100）
- **保留**：所有 code-behind 引用的 `x:Name`（SearchBox、RecordCountRun、AddDialogOverlay）、所有 Click 事件处理器
- **编译**：0 错误，仅预存 warnings（SQLitePCLRaw NU1903 + NewQuotationViewModel CS8601）

  - 总金额 StackPanel 内两个 TextBlock 各加 `HorizontalAlignment="Center"`
