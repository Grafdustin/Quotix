## QuotixDesktop — WPF 桌面报价系统

**将 Quotix (D:\桌面\Quotix) 的报价系统功能迁移为 C# WPF 桌面程序。**

### 技术栈
- .NET 10 WPF + XAML
- CommunityToolkit.Mvvm (MVVM 源码生成器)
- Microsoft.Data.Sqlite (复用原 quotation.db)
- ClosedXML (Excel 导出/导入)

### 项目位置
- 源码: `D:\桌面\QUO\QuotixDesktop\`
- 发布: `D:\桌面\QUO\QuotixDesktop\publish\QuotixDesktop.exe`

### 功能模块
1. **新建报价** — 公司/客户信息 + 报价项目明细 + 单价计算器 + 保存/更新/导出 Excel
2. **报价历史** — 列表查看/编辑/导出/删除
3. **产品数据库** — NDT(价表/货期) + RVI(Change/OT Code)，支持 Excel 导入和搜索
4. **表头数据库** — 负责人信息管理 + 客户信息管理

### 架构
- Models: Quotation, QuotationItem, Product, Owner, Customer
- Services: DatabaseService, QuotationService, ProductService, HeaderService, ExportService
- ViewModels: MainViewModel, NewQuotationViewModel, HistoryViewModel, ProductDatabaseViewModel, HeaderDatabaseViewModel
- Views: MainWindow.xaml, NewQuotationView, HistoryView, ProductDatabaseView, HeaderDatabaseView, PriceCalculatorDialog

### Bug 修复 — 程序启动崩溃 + 数据库迁移 LiteDB (13:10)
- **崩溃根因**: NewQuotationView.xaml 第391行 `{Binding QuickSearchResults.Count}` — ObservableCollection.Count 是只读属性，WPF Run.Text 默认 TwoWay 绑定导致 InvalidOperationException。修复：加 `Mode=OneWay`。
- **崩溃 #2**: UpdateQuickDBButtons() 中 VM 属性在 DataContext 未设置时访问导致 NullReferenceException。修复：加 DataContext 空检查。
- **数据库迁移**: SQLite → **LiteDB**（纯 C# 嵌入式 NoSQL，零原生依赖）
  - 原因：SQLite 需要 SQLitePCLRaw native DLL，启动/检索慢，几千条数据体验差
  - LiteDB 优势：纯 C#、LINQ 强类型查询、BSON 文档直存（去 JSON 序列化开销）、自动索引、零连接管理
  - Product.Data 从 JSON 字符串 → BsonDocument（直接遍历，无需 `JsonDocument.Parse`）
  - 启动时自动从旧 SQLite 迁移数据（检查 quotation-v2.db 不存在 + quotation.db 存在）
  - 旧 SQLite 数据库备份为 quotation.db.bak
  - 新增 LiteDB NuGet 包，所有 Service 层重写（去 SQL 拼接，改用 LINQ/Collection API）
  - Models 加 `[BsonId]` 属性

### Bug 修复 — 快速输入卡死 (13:37)
- **三重根因**: 
  1. 零防抖 — 每按一个键都发全量搜索（几千条 BsonDocument 遍历）
  2. 无缓存 — 每次击键都从 LiteDB 重新加载全部产品
  3. 无取消 — 旧搜索没跑完新搜索已发起，请求堆积
- **修复**:
  1. **250ms 防抖** — 用 `CancellationTokenSource` + `Task.Delay(250, token)`，用户停止输入后才触发搜索
  2. **搜索索引缓存** — `BuildSearchIndex()` 将全部产品预计算为 `QuickSearchIndex`（SearchText 小写拼接 + Title/Subtitle/Price/RawData），首次加载后缓存，后续击键只做内存 List 过滤
  3. **取消令牌** — 每次新输入 `CancelSearch()` 取消前次
  4. **一次性 BsonDocument 遍历** — 索引构建时用 `StringBuilder` 一次遍历完成 SearchText/Title/Subtitle/Price/RawData 全部提取
  5. **启动安全** — PositionQuickSearchPopup 加 null 检查，TextChanged/GotFocus 加 `IsLoaded` 检查防止初始化时误触发
- 同时修复 Save/Export/PriceCalculator 方法适配 LiteDB 模型 API（TotalAmount→TotalAmount, TotalPrice→TotalPrice, ExportExcel→ExportToExcel）

### Bug 修复 — 程序启动无界面 (14:32)
- **现象**: 双击 QuotixDesktop.exe 后台有进程但无窗口显示
- **根因**: App.xaml 引用 `<ResourceDictionary Source="Themes/LightTheme.xaml"/>`，但 .csproj 中 Themes XAML 文件标记为 `<None>` 而非 `<Content>`，导致 publish 时未复制到输出目录。WPF 在 `InitializeComponent()` 阶段加载主题文件失败抛出 XamlParseException，此时 `OnStartup` 尚未执行（异常处理未注册），程序静默崩溃
- **修复**:
  1. `.csproj` 中 `Themes/LightTheme.xaml` 和 `DarkTheme.xaml` 从 `<None Update>` 改为 `<Content Include>` + `CopyToOutputDirectory=PreserveNewest`
  2. `App.xaml.cs` 添加 `DispatcherUnhandledException` + `AppDomain.UnhandledException` 全局异常捕获，写入 `%LocalAppData%\Quotix\error.log`
- **验证**: dotnet run 正常启动，publish 单文件包含 Themes 目录，140MB+ 可执行

### Bug 修复 — 程序启动无界面 #2 (14:36)
- **现象**: 用户反馈程序打不开，只有后台进程，窗口不显示
- **排查**: 无 error.log、编译 0 错误
- **根因**: `App.xaml` 缺少 `StartupUri="MainWindow.xaml"` 属性。WPF 应用启动时，没有 `StartupUri` 且 `OnStartup` 只调了 `base.OnStartup(e)`（不创建窗口），导致进程在后台运行但没有窗口渲染
- **修复**:
  1. `App.xaml` 添加 `StartupUri="MainWindow.xaml"`
  2. `App.xaml.cs` 的 `DispatcherUnhandledException` 中移除 `Current.Shutdown()` 并改为 `MessageBox.Show` 提示用户，避免运行时异常静默崩溃
- **验证**: `dotnet build` 编译成功，0 错误

### Bug 修复 — WPF XAML 转换器无法加载 / DB Schema 不兼容 (14:45)
- **现象**: StartupUri 加好后，报 `XamlParseException: 无法加载未知类型 BoolToVisibilityConverter`
- **根因 #1**: 外部 ResourceDictionary XAML 文件运行时加载时，`clr-namespace` 缺少 `;assembly=` 限定符，导致 WPF 无法在运行时解析 CLR 类型
- **修复 #1**: `LightTheme.xaml` 和 `DarkTheme.xaml` 的 `xmlns:conv` 改为 `clr-namespace:QuotixDesktop.Converters;assembly=QuotixDesktop`
- **根因 #2**: 修复 #1 后暴露 — `quotation.db` 是旧 schema，`products` 表缺少 `table_name`/`created_by`/`created_at`/`updated_at` 列，`CREATE INDEX IF NOT EXISTS` 在列不存在时仍抛 `SqliteException: no such column: table_name`
- **修复 #2**: `DatabaseService.EnsureTables()` 重构为三步：
  1. `CREATE TABLE IF NOT EXISTS`（幂等建表）
  2. `TryExecuteNonQuery` 逐个 `ALTER TABLE ADD COLUMN`（补旧表缺失列，错误忽略）
  3. `TryExecuteNonQuery` 逐个 `CREATE INDEX IF NOT EXISTS`（建索引，错误忽略）
- **增强 #3**: `App.xaml.cs` 的 `LogException` 改为递归记录 `InnerException` 链；异常弹窗展示最内层错误消息
- **验证**: clean build + dotnet run，进程正常运行不崩溃
