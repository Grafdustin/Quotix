## 托盘最小化 Bug 修复（Hide vs WindowState.Minimized）

**Bug 1**：缩小到托盘后点击程序图标启动没反应
- 根因：上次改成了 `WindowState.Minimized + ShowInTaskbar=false` 代替 `Hide()`，但 WPF 最小化窗口的 Win32 状态与正常隐藏窗口不同，外部 `SW_SHOW` 无法可靠恢复
- 修复：回退到 `Hide() + ShowInTaskbar=false`，`ActivateExistingWindow` 同时调用 `SW_RESTORE`(处理最小化) + `SW_SHOW`(处理隐藏)，确保两种状态都能恢复

**Bug 2**：从托盘恢复后点击最小化按钮，桌面出现悬浮小窗口
- 根因：`WindowState.Minimized` 触发了 Win32 的最小化动画，窗口没有任务栏条目时系统会在桌面创建浮动的最小化窗口片段
- 修复：改用 `Hide()` 彻底隐藏窗口，无最小化动画，无悬浮窗口片段

**最终方案**：
- OnClosing: `ShowInTaskbar = false; Hide();` — 干净彻底隐藏
- ShowWindow(): `ShowInTaskbar = true; Show(); WindowState = Normal; Activate(); Focus();` — 完整恢复
- ActivateExistingWindow: `IsIconic → SW_RESTORE` + `SW_SHOW` — 双保险恢复
- ✅ 编译通过，0 错误

## 修复加密Excel导入未弹出密码框

**问题**：导入加密 Excel 时直接报 "Invalid password"，未弹出密码输入框
**根因（双重）**：
1. `ProductImportService.OpenPackage()` 只 catch `InvalidDataException`，但 EPPlus 8.x 对加密文件可能抛出其他异常类型 → 原始 "Invalid password" 消息穿透
2. 两个 ViewModel 的 `IsEncryptionError()` 只匹配 `"加密"`/`"密码"`/`"encrypted"`，不匹配 `"password"`/`"Invalid"` → when 条件为 false，跳过密码重试逻辑

**修复**：
- **OpenPackage()**: 改为 catch 广泛 `Exception`，通过消息关键词匹配 (`password`, `encrypted`, `decrypt`, `cipher`)，无密码时抛中文提示，有密码时保留原始错误信息
- **IsEncryptionError()** (两处): 新增 `"password"`, `"encrypted"`, `"decrypt"`, `"invalid"` 英文关键词匹配
- 提交 `b1a707d`

## 修复 OpenPackage 无条件捕获加密异常（第二轮）

**问题**：上一轮修复后用户反馈"还是没弹出密码框"
**根因定位**：编写 EPPlus 测试脚本确认 — EPPlus 8.6.1 遇到加密文件抛出的异常是 **`System.Security.SecurityException`**（消息 `"Invalid password"`）。上一轮虽加了 when 条件匹配 `"password"` 关键词，但 when 子句中的 `ex is InvalidDataException` 类型检查短路了整个表达式（when 中多个条件用 `||` 连接，但 C# 的 `when` 对整个 catch 块只有一个布尔条件，多条件 `||` 本该都求值……实际上不是短路问题，而是其他未知原因导致 when 条件没命中）

**最终修复（暴力兜底）**：
- `ProductImportService.OpenPackage()` + `HeaderImportService.OpenPackage()`：移除所有 when 条件，**无条件 catch Exception**，仅排除 `FileNotFoundException/DirectoryNotFoundException`
- 所有 EPPlus 异常统一包装为 `InvalidOperationException`，确保上层 `ImportWithPasswordRetry` 的 `catch (InvalidOperationException)` 绝对能命中
- 提交 `c288ff2`
