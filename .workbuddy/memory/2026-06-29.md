## 🔴 弹窗不显示修复 — DialogService PushFrame 嵌套消息冲突
- **问题**：所有弹窗无反应、不显示，WPF-UI MessageBox 窗口从未创建
- **根因**：原 `DialogService.Show()` 使用 `Dispatcher.InvokeAsync(() => await msgBox.ShowDialogAsync())` + `Dispatcher.PushFrame(frame)` 嵌套消息循环模式，与 WPF-UI MessageBox 内部 `ShowDialogAsync` 的异步创建流程冲突（嵌套 PushFrame 无法正确泵送 async 状态机延续）
- **修复**：改为 `Dispatcher.Invoke(() => { msgBox = new(); msgBox.ShowDialog(); })` 同步模式
  - `Window.ShowDialog()` 自带嵌套消息泵（WPF 框架级），无需手动 PushFrame
  - 通过 `msgBox.DialogResult` 判定按钮点击：`true`→Primary, `false`/`null`→None
  - 移除 `System.Windows.Threading` 引用
- **编译**：0 错误

## 全量导出算术溢出修复
- **报错**：`导出失败: Arithmetic operation resulted in an overflow`
- **根因**：`EncryptToFile` 将 JSON → GZip → AES 全部在内存中完成，创建 3 个巨大 byte[]（压缩后 + 加密后 + 写入前），大表导出时 `MemoryStream.ToArray()` 溢出 int 范围
- **修复 V1**：`EncryptToFile` 重写为流式管道 — `StreamWriter → GZipStream → CryptoStream → FileStream`，数据边压缩边加密边写入磁盘，零中间大数组
- **仍然失败**：溢出发生在更上游的 `JsonSerializer.Serialize(backup)` — 大产品库的 `DataJson` 字段序列化成单一 JSON 字符串时溢出
- **修复 V2**：新增 `EncryptToFile(string, FullBackupData, string)` 方法，用 `Utf8JsonWriter` 直接写入 GZip → Crypto → File，零字符串中间态
  - 管道：`JsonSerializer → Utf8JsonWriter → GZipStream → CryptoStream → FileStream`
  - 全程不产生 string 或巨大 byte[]，仅在 Buffer 级处理
- `DecryptFromFile` 无需修改（文件格式 magic|salt|iv|cipher 完全兼容新老格式）
- **编译**：0 错误

## 全量导出 GZipStream 算术溢出修复（最终修复）
- **报错**：`导出失败: Arithmetic operation resulted in an overflow`
- **根因**：GZipStream 内部用 `Int32` 跟踪未压缩数据总字节，用于写 GZIP 尾部 ISIZE 字段。数据超过 ~2GB 时 int 溢出 → OverflowException（.NET Framework 已知限制）
- **修复**：`CryptoService.EncryptToFile(FullBackupData)` 中 `GZipStream` → `DeflateStream`
  - Deflate 是同样压缩算法但无 GZIP 包装头/尾（CRC32 + ISIZE），不存在大小限制
  - 加密文件无需外部工具读取，不需要 GZIP 格式
- **兼容性**：`DecryptFromFile` 新增 `TryDecompress()` 方法 — 先试 GZip 再试 Deflate，旧备份仍可正常导入
- **编译**：0 错误
